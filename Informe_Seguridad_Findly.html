Informe de Seguridad y Privacidad de Datos - Findly

Este informe detalla las razones técnicas por las cuales la aplicación Findly no compromete la integridad ni la privacidad de los datos de la empresa, garantizando que la información sensible nunca sale del entorno local.

1. Arquitectura de "Frontend Desacoplado"

Aunque la interfaz de usuario está alojada en Netlify, esta actúa únicamente como un "contenedor vacío" o una "plantilla visual".

  • Netlify no procesa datos: Los servidores de Netlify solo entregan los archivos necesarios para mostrar la web.
  • Sin base de datos externa: No existe ninguna base de datos en la nube donde se almacenen referencias o stocks.

2. Procesamiento de Datos 100% Local

La aplicación utiliza un modelo de comunicación punto a punto entre el navegador del usuario y un servidor local dentro de la red de la empresa.

  • Servidor Backend Local: Todas las operaciones se realizan en la dirección IP privada 172.30.0.161.
  • Rutas de Red Internas: El servidor accede a archivos de Excel locales (ej. \\172.30.0.10\Logistica\...). Estos archivos nunca se suben a internet.
  • Carga de Archivos: Los archivos de pedido se envían directamente al servidor interno, sin pasar por servidores externos.

3. Aislamiento de Internet

  1. Sin Exfiltración: El código no contiene llamadas a APIs de terceros ni servicios de análisis.
  2. Operación en Red Privada: La lógica de negocio reside totalmente dentro de la red de la oficina.
  3. Privacidad por Diseño: Los datos se procesan en la memoria del servidor local y permanecen bajo el control de la empresa.

4. Conclusión

La aplicación Findly visualiza datos de forma segura, manteniendo una barrera infranqueable hacia internet. La información de negocio permanece en sus propios servidores.
